    HTML, CSS, Javascript 작성 시 W3C의 웹 규격에 맞는 코딩을 함으로써 어느 브라우저, 기기에서 사이트가 의도된 대로 보여지고 작동되는 기법
  • 왜 크로스 브라우징 작업이 필요할까 ?
    우리가 쓰는 크롬이나 익스플로어 외에도 여러 브라우저 마다 랜더링 엔진이 다르기 때문이다
    이 브라우저들은 각각 다른 글꼴이나 속성을 가지고 있어 동일한 코드를 작성해도 브라우저마다 화면이 다르게 보일 수 있다.
    • 작동되지 않는 HTML5, Javascript 코드가 존재
    • 해석하지 못하는 CSS 코드 존재
    • 브라우저 버그들이 존재
    • 브라우저마다 자체적인 CSS 스타일
  • 크로스 브라우징을 하려면 ?
    전반적으로 브라우저에 사용되는 검사기준을 파악해야하며, 웹표준, 웹접근성에 대한 지식을 가지고 있어야한다.
    W3C통합 검사 사이트를 통해 개발이 된 사이트가 웹 표준에 맞게 개발되었는지 확인할 수 있다.

웹접근성

  • 웹 접근성(Web Accessibility) 장애인, 고령자 등이 웹 사이트에서 제공하는 정보에 비장애인과 동등하게 접근하고 이해할 수 있도록 보장하는 것
    비장애인이 웹상에서 제공되는 텍스트와 이미지, 영상 등을 접했을 경우, 한눈에 재빨리 내용 파악이 가능하지만, 장애인은 그렇지 않다. 그림이나 사진들을 제공할 때 눈으로 볼 수 없는 경우를 대비하여 그림이나 사진을 대신 할 수 있는 설명을 텍스트로 제공해야 하며, 동영상이나 오디오의 경우 청각장애인을 위한 음성정보를 문자로 제공해야 합니다. 또한, 마우스를 사용할 수 없는 사용자를 위하여 키보드만으로도 모든 콘텐츠에 접근하여 이용할 수 있도록 해야 하며, 움직임이 느린 사용자를 위해 시간조절기능을 제공해야 한다.
  • 웹 접근성의 필요성
    • 누구에게나 동등한 기회 제공을 위해 웹 접근성 준수 필요
      웹 접근성을 준수하게 되면 장애인, 고령자 등이 웹 사이트에서 제공하는 정보에 비장애인과 동등하게 접근하고 이용할 수 있게 된다 현대사회에서 웹은 교육, 고용, 정부, 전자상거래, 건강, 여가 등 삶의 여러 가지 측면들에서 점점 더 중요한 자원으로 활용되고 있다
    • 웹 접근성 보장은 법률에 명시된 의무사항
      웹 접근성 보장은 「국가정보화기본법」과 「장애인차별금지 및 권리구제 등에 관한 법률(이하 "장애인차별금지법")」등 법률에 명시된 의무사항이다 특히 장애인차별금지법은 행위자에 대한 단계적 범위를 명확히 하였으며, 불이행시 처벌 등이 구체적으로 성문화된 법이다

CSS의 사이즈 단위

  • px : 가장 기본적으로 사용되는 단위. / 절대 단위 픽셀단위라는 고정값에 따라 정해지기 때문에 화면의 크기나 확대와 같이 사용자가 임의로 정의할 수 없는 고정된 값.
  • em : em은 부모 요소를 기준으로 자식 요소의 크기를 정하는 것.
    (ex)
    <div class= "parent">
       <div class= "child"></div>
    </div>

    .parent{font-size: 12px;}
    .child{font-size: 1.2em;}

    다음과 같이 부모 요소를 12px로 정의했을 때, 자식 요소는 12px*1.2=14.4px로 정의됩니다.
  • rem : rem은 root em의 약자. 가장 최상단(root) 최상위 태그 html에 정의된 사이즈를 기준으로 배수하겠다는 것을 의미
    (ex)
    html{font-size:12px;}
    div{font-size:1.2rem;}

    최상위 요소를 기준으로 한 1.2*12px의 사이즈를 갖는다
    rem 시스템은 모바일 환경에서 접근성을 위해 글자 크기를 키우거나 그리드 시스템에 사용하기에 유용하다
  • %(백분율) : % 단위는 부모 요소를 기준으로 비율을 표현된다
    body{font-size:14px;}
    div{font-size:120%; /* 14px * 1.2 = 16.8px */
        padding: 2em; /* 16.8px * 2 = 33.6px */ }
  • vh/vw : vh와 vw는 각각 viewport height, viewport width의 약자 Viewport란? 전체 웹페이지 가운데 브라우저 창에 보이는 영역
    이것들은 뷰포트(화면의 크기)의 높이와 너비에 비례한다
    이 단위는 반응형 페이지를 만들 때 아주아주 유용하게 사용될 수 있다

모바일웹

  • 모바일 웹이란? 데스크톱 PC용 웹사이트와 별개로 모바일 브라우저에서 이용자들이 편하게 이용할 수 있도록 PC용 웹사이트를 모바일 스크린 크기에 맞춰 줄여 놓은 것을 뜻한다 예를 들면, 네이버나 다음과 같은 포털 사이트를 스마트폰으로 접속하면 URL 주소가 m.naver.com 혹은 m.daum.net 과 같이, 같은 사이트인데 PC용 웹사이트와 모바일용 웹사이트, 이렇게 다른 두 가지 버전의 웹사이트가 존재하는 것이다
  • 모바일 웹 제작시 주의사항
    • viewport 태그를 포함해야한다
      <meta name="viewport" content="width=device-width, initial-scale=1.0 user-scalable="no">
      • width=device-width : 브라우저 가로사이즈를 기기 가로사이즈에 맞춤
      • initial-scale=1.0" : 초기확대값
      • user-scalable="no" : 사용자 확대 축소 가능
    • 웹 페이지의 너비가 변형되어야한다.
      다양한 종류와 너비의 디바이스에서 모두 가득판 화면이 구성되어야한다. %와 px이 조화롭게 표현되어야 한다.
    • 가변형 레이아웃
      모든 컨텐츠의 사이즈는 %로만 이루어진 않고있다
      px,rem과 %를 함께 사용하는 경우도 있다.
    • 최소값과 최대값
      이미지 또는 컨텐츠의 크기가 최소값과 최대값이 있어야 한다
      • width or height
      • min-width or min-height
      • mix-width or mix-height
    • 모바일 전용소스
      모바일에 특화된 요소를 적극 활용하자
      • <a href = "tel: 01012345678">전화걸기</a>
      • <a href = "sms: 01012345678">문자보내기</a>
      • <a href = "mailto: abc@google.com">메일보내기</a>
      • <input type="number"> 등등
    • 기기호환성
      현실적으로 모든 기기에 대응할 수 없기에 클라이언트와 협의하여 적용기기의 범위를 선정해야한다

반응형

  • 반응형이란? 한 가지의 웹사이트로 다양한 종류의 기기에 최적화된 화면을 보여주는 것
    반응형 웹은 동작하는 기기의 화면과 해상도에 맞춰 저절로 반응하여 변한다. 모바일 웹이 PC버전의 홈페이지와 모바일 홈페이지를 따로 제작하는 것이라면, 반응형 웹은 하나의 홈페이지를 구축하면 데스크탑, 태블릿, 그리고 모바일에서도 최적화된 화면으로 볼 수 있다
  • 반응형웹 기술이 필요한 이유 반응형 웹은 웹사이트를 이용하는 사람들에게 모든 기기에서 최적화된 웹사이트를 제공할 수 있고, PC 버전의 웹사이트와 모바일 버전의 웹사이트 두 가지 모두를 만들지 않아도 됨에 따라 비용과 시간 그리고 인력을 줄일 수 있다 또한, 모바일웹 접속률이 많이 늘어난 만큼 모바일 버전 또한 신경써주어야 한다
  • 반응형의 장점
    • 간편한 유지보수
      pc , mobile 전용 사이트를 따로 구축하는 것보다 하나의 html,css로 운영하는것이 편리하다
    • 즉시반응하는 레이아웃
      기기의 화면이나 해상도에 구애받지 않고 웹사이트의 레이아웃 구성이 달라지며, 화면의 변화 브라우저의 사이즈 등에 즉시 반응한다
    • 검색엔진 최적화
      pc와 mobile을 구분하여 제작할 경우, 비슷한 도메인에 내용도 비슷하여 검색결과에 제외시키거나 상위권에 배치시키는 것이 상대적으로 불리하지만 반응형으로 제작할 경우 하나의 url을 사용하기 때문에 검색과 노출에 용이하다
  • 반응형의 단점
    • 길어진 로딩시간
      페이지가 로딩될때 모든 html,css,js를 불러와 필요없는 리소스까지 로딩되는 현상이 일어난다 그렇기 때문에 하나의 화면에 많은 컨텐츠를 표현해야 할 때에는 적합하지 않다 네이버와 같은 대형 포털 사이트나 인터넷 쇼핑몰처럼 한 화면에 많은 컨텐츠를 보여줘야 하는 사이트를 반응형 웹으로 구축하면, 개발하기도 복잡하고 한 번에 많은 양의 컨텐츠를 불러와야 하기 때문에 로딩 속도도 느려지는 단점이 있다 그렇기 때문에 반응형 웹은 작은 규모의 사이트를 만들 때 더 효율적이다
    • 한정적 지원
      IE10이상에서만 지원하는 기술이기 때문에 그 이하의 브라우저 사용자들은 제약이 된다
  • 반응형 웹 제작방법
    • css 내부에 @media query 사용하기
      (ex)
      @media (max-width:1000px) { /* width가 1000px 까지의 css 내용 */ }
    • 외부 링크로 사용하기
      <link rel="stylesheet" media="(min-width:771px) and (max-width:1200px)" href="css/index_tablet.css">
      /* 771 ~ 1200px 사이의 미디어쿼리 */
      <link rel="stylesheet" media="(min-width:0px) and (max-width:770px)" href="css/index_mobile.css">
      /* 0 ~ 770px 사이의 미디어쿼리 */
